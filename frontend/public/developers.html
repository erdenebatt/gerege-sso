3. OAuth2 Authorization Flow

1

User clicks "Sign in with Gerege" — your app redirects to:

GET https://sso.gerege.mn/api/oauth/authorize ?client_id=YOUR_CLIENT_ID &redirect_uri=https://yourapp.com/callback &response_type=code &scope=openid+profile &state=RANDOM_CSRF_TOKEN
2

User sees consent screen — Gerege SSO shows "{Your App Name} байгууллагад нэвтрэх гэж байна" with the list of data being shared. User clicks "Зөвшөөрөх".

3

Gerege redirects back with an authorization code:

https://yourapp.com/callback?code=AUTH_CODE&state=RANDOM_CSRF_TOKEN
4

Your server exchanges the code for a token:

POST https://sso.gerege.mn/api/oauth/token Content-Type: application/x-www-form-urlencoded grant_type=authorization_code &code=AUTH_CODE &redirect_uri=https://yourapp.com/callback &client_id=YOUR_CLIENT_ID &client_secret=YOUR_CLIENT_SECRET
5

Receive an enriched JWT:

{ "access_token": "eyJhbGciOiJI...", "token_type": "bearer", "expires_in": 3600 }

4. JWT Payload

The decoded JWT contains user identity data from Gerege:

{ "iss": "https://sso.gerege.mn", "sub": "40555880567", "aud": ["your-client-id"], "exp": 1770305833, "iat": 1770302233, "email": "user@gmail.com", "picture": "https://lh3.googleusercontent.com/...", "email_verified": true, "gerege": { "gen_id": "40555880567", "family_name": "Боржигон", "last_name": "Эрдэнэ", "first_name": "Бат", "birth_date": "1990-05-20", "gender": "male" } }

Gerege Claim Fields

FieldTypeDescription
gen_idstringUnique 11-digit Gerege ID (always present)
family_namestringУргийн овог (clan/family name)
last_namestringОвог (surname)
first_namestringНэр (given name)
birth_datestringYYYY-MM-DD format
genderstring"male" or "female"

Note: If the user has not completed identity verification, only gen_id will be present in the gerege object. The fields civil_id and reg_no are never included.

5. API Reference

Authorization GET

GET /api/oauth/authorize
ParameterRequiredDescription
client_idYesYour registered client ID
redirect_uriYesMust match registered URI exactly
response_typeYesMust be code
scopeNoDefault: openid profile
stateRecommendedCSRF protection token

Token Exchange POST

POST /api/oauth/token
ParameterRequiredDescription
grant_typeYesMust be authorization_code
codeYesAuthorization code from callback
redirect_uriYesSame URI used in authorize request
client_idYesYour client ID
client_secretYesYour client secret

6. Getting Started

To register your application as an OAuth2 client, contact the Gerege SSO administrator. You will receive:

  • client_id — your public application identifier
  • client_secret — keep this secure, never expose in frontend code

Provide the following when registering:

  • App name — shown on the consent screen (e.g. "Golomt Bank")
  • Redirect URI — where users are sent after authorization
