3. OAuth2 зөвшөөрлийн урсгал

1

Хэрэглэгч "Gerege-ээр нэвтрэх" товч дарна — таны апп дараах хаягруу чиглүүлнэ:

GET https://sso.gerege.mn/api/oauth/authorize ?client_id=YOUR_CLIENT_ID &redirect_uri=https://yourapp.com/callback &response_type=code &scope=openid+profile &state=RANDOM_CSRF_TOKEN
2

Хэрэглэгч зөвшөөрлийн дэлгэцийг харна — Gerege SSO "{Таны аппын нэр} байгууллагад нэвтрэх гэж байна" гэж харуулаад, хуваалцах мэдээллийн жагсаалтыг үзүүлнэ. Хэрэглэгч "Зөвшөөрөх" товч дарна.

3

Gerege буцаан чиглүүлнэ — зөвшөөрлийн кодтой хамт:

https://yourapp.com/callback?code=AUTH_CODE&state=RANDOM_CSRF_TOKEN
4

Таны сервер кодыг токен руу солино:

POST https://sso.gerege.mn/api/oauth/token Content-Type: application/x-www-form-urlencoded grant_type=authorization_code &code=AUTH_CODE &redirect_uri=https://yourapp.com/callback &client_id=YOUR_CLIENT_ID &client_secret=YOUR_CLIENT_SECRET
5

Баяжуулсан JWT токен хүлээн авна:

{ "access_token": "eyJhbGciOiJI...", "token_type": "bearer", "expires_in": 3600 }

4. JWT агуулга

Задалсан JWT нь Gerege-ийн хэрэглэгчийн мэдээллийг агуулна:

{ "iss": "https://sso.gerege.mn", "sub": "40555880567", "aud": ["your-client-id"], "exp": 1770305833, "iat": 1770302233, "email": "user@gmail.com", "picture": "https://lh3.googleusercontent.com/...", "email_verified": true, "gerege": { "gen_id": "40555880567", "family_name": "Боржигон", "last_name": "Эрдэнэ", "first_name": "Бат", "birth_date": "1990-05-20", "gender": "male" } }

Gerege claim талбарууд

ТалбарТөрөлТайлбар
gen_idstring11 оронтой өвөрмөц Gerege дугаар (үргэлж байна)
family_namestringУргийн овог
last_namestringОвог
first_namestringНэр
birth_datestringYYYY-MM-DD формат

Анхаар: Хэрэв хэрэглэгч иргэний баталгаажуулалт хийгээгүй бол gerege объектод зөвхөн gen_id байна. civil_id болон reg_no талбарууд гуравдагч талын токенд хэзээ ч орохгүй.

5. API лавлагаа

Зөвшөөрөл GET

ПараметрЗаавалТайлбар
client_idТиймБүртгэгдсэн client ID
redirect_uriТиймБүртгэлтэй URI-тай яг таарах ёстой

Токен солилцоо POST

POST /api/oauth/token
ПараметрЗаавалТайлбар
grant_typeТиймauthorization_code байх ёстой
codeТиймCallback-аас ирсэн зөвшөөрлийн код
redirect_uriТиймAuthorize хүсэлтэд ашигласан URI-тай ижил байх
client_idТиймТаны client ID
client_secretТиймТаны client secret

6. Эхлэх заавар

Өөрийн аппликэйшнийг OAuth2 клиентээр бүртгүүлэхийн тулд Gerege SSO админтай холбогдоно уу. Та дараах мэдээллийг хүлээн авна:

  • client_id — таны аппликэйшний нийтийн танигч
  • client_secret — нууц түлхүүр, frontend кодонд хэзээ ч ил бичиж болохгүй

Бүртгүүлэхдээ дараах мэдээллийг өгнө үү:

  • Аппын нэр — зөвшөөрлийн дэлгэцэнд харагдана (жнь: "Голомт банк")
  • Redirect URI — зөвшөөрлийн дараа хэрэглэгчийг чиглүүлэх хаяг
